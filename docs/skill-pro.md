## 3. 浏览市场

用户在这里浏览社区贡献的技能，一键安装。

### 3.1 搜索 + 筛选栏

**排行榜（无搜索时）：** 三个按钮切换——All Time / Trending / Hot。每个榜的数据来源不同。

**搜索框：**
- 左侧放大镜图标
- 占位文字："搜索 skills.sh..."
- 用户打字后自动搜索
- 搜索时隐藏排行榜 tab，只显示搜索结果
- 搜索结果按安装量从高到低排列

**Source 过滤器：**
- 标签"Source:"后面跟着一排圆角标签按钮
- 第一个是 [All]，后面每个 `@用户名/仓库名`
- 点击某个 source 就只看那个贡献者的技能
- 如果 source 太多一行显示不下，自动折叠成 [...] 按钮
- 点 [...] 弹出面板，里面可以搜索 source 名称

### 3.2 技能卡片列表

每条技能一张卡片，网格排列（桌面 3 列，窄屏 2 列）。

**卡片内容：**

```
┌──────────────────────────────┐
│ [头像] 技能名         [🔗] [＋] │
│        @来源   1.2K 安装      │
└──────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 头像 | 贡献者的 GitHub 头像，圆形 |
| 技能名 | 技能显示名称 |
| skill_id | 当技能名和目录名不同时，在名称下方用小字显示 |
| 来源标签 | `@用户名/仓库名`，点击可筛选该来源 |
| 安装量 | 仅 All Time 榜显示，超过 1000 显示为 1K，超过 100 万显示为 1M |
| 外部链接 | 打开该技能在 skills.sh 的页面 |
| 安装按钮 | 未安装显示 `＋`；已安装显示绿色 `✓` "已安装" |
| 安装中 | 显示旋转动画 + "取消" 按钮 |

**已安装标记：** 已安装的技能卡片上显示绿色对勾，安装按钮消失。

### 3.3 安装操作

点击 `＋` → 系统开始安装 → 按钮变为旋转动画 + 红色"取消"按钮。

安装过程中：
1. 克隆阶段（从 GitHub 拉取）
2. 安装阶段（复制到本地 skill 库）
3. 完成 → 弹出通知"安装成功"

通知中包含"查看"按钮，点击跳转到已安装技能的详情页。

如果安装失败：通知变为错误提示。

**取消安装：** 安装中点击红色"取消"按钮可中断。

### 3.4 分页与加载更多

- 每页 24 条
- 底部分页导航：上一步 / 页码 / 下一步
- 搜索模式下底部还有"加载更多结果"按钮
- 页码过多时自动折叠，显示首尾 + 当前附近页码

### 3.5 无结果

搜索无结果时显示空状态：一个大搜索图标 + "没找到相关 skills" + "试试换个关键词搜索"。

### 3.6 搜索防抖

用户停止输入约半秒后才发起搜索，避免频繁请求。

## 4. 本地安装

用户从电脑本地选择 skill 文件或文件夹进行安装。

### 4.1 操作入口区域

```
📁 本地安装
从本地文件夹或压缩包安装 skill

[📁 选择文件夹] [📦 选择压缩包] [📥 批量导入]
```

| 按钮 | 做什么 |
|------|--------|
| 选择文件夹 | 打开系统文件夹选择器，选一个 skill 目录 |
| 选择压缩包 | 打开文件选择器，选 .zip 或 .skill 文件 |
| 批量导入 | 选择一个文件夹，系统自动扫描其中所有 skill 并批量导入 |

### 4.2 安装流程

用户选择源 → 弹出"正在安装"通知 → 安装完成 → 通知变为"安装成功"（含"查看"按钮）→ 列表刷新。

如果失败：通知变错误提示，页面顶部显示红色错误横幅（含"重试"按钮）。

### 4.3 批量导入流程

选择文件夹 → 逐条处理 → 实时通知当前进度（"正在处理第 3/15 个：xxx"）→ 完成后汇总结果：

- 全部成功："已导入 N 个，跳过 M 个"
- 部分错误："N 个导入失败" + 错误详情
- 全部跳过："全部已导入或跳过"

### 4.4 发现已有技能

进入本地安装页时，系统自动扫描设备上各 agent 工作空间中已存在但尚未导入的 skill。

**扫描结果显示：**

```
已发现的 Skills               [🔄 重新扫描] [📥 全部导入]
扫描了 N 个工具，发现 M 个 skill
```

列表中的每一项：

| 元素 | 说明 |
|------|------|
| 技能名 | 可点击铅笔图标重命名（内联编辑） |
| 已导入 | 绿色标签，表示已在中心库中 |
| 位置数 | "N 个位置"表示该 skill 出现在几个工具的工作空间里 |
| 发现日期 | 日历图标 + 日期 |
| 所属工具 | 小标签显示哪个 agent |
| 路径 | 代码字体显示文件路径 |
| 导入按钮 | 逐个导入 |
| 全部导入 | 页面顶部按钮，一次导入所有 |

**重命名交互：** 点击铅笔图标 → 名称变为输入框 → 输入新名称 → Enter 确认 / Escape 取消。

**空状态：** 没发现任何 skill 时显示文件夹搜索图标 + "没找到可导入的 skill" + 提示文字。

## 5. 从 Git 安装

用户从 Git 仓库 URL 克隆安装。

### 5.1 输入区域

```
🐙 从 Git 仓库安装
从 GitHub 等 Git 仓库安装 skill

Git 仓库 URL
[________________________________]

[⬇ 克隆并安装]
```

- 支持按 Enter 键触发安装
- 如果输入的仓库已安装过，在输入框下方显示黄色提示"该仓库已安装为 xxx"，按钮文字变为"重新安装"

### 5.2 预览对话框

克隆到临时目录后，弹出选择对话框：

```
选择要安装的 skill          [✕]
从仓库中发现以下 skill 配置：

[全选] · [取消全选]

☑ [技能名可编辑________]
   技能描述文字...
☐ [技能名可编辑________]
   技能描述文字...

         [取消]   [⬇ 安装选中]
```

- 每条 skill 可勾选/取消
- 技能名可内联编辑
- "全选"/"取消全选" 快捷操作
- 无 skill 时展示空状态提示
- 安装过程中按钮显示 spinner

## 6. 交互通用规则

### 6.1 通知体系

所有通知出现在右下角。

| 类型 | 外观 | 何时出现 |
|------|------|---------|
| 进行中 | 带旋转图标 | 正在安装/导入 |
| 成功 | 绿色 | 操作完成 |
| 失败 | 红色 | 操作出错 |
| 取消 | 默认色 | 用户主动取消 |

**成功通知的特殊行为：** 含"查看"按钮，点击可跳转到已安装技能的详情页面。

### 6.2 异步操作（所有写操作）

用户的任何安装、导入、删除请求，都不会阻塞界面。

1. 用户点击 → 通知显示"操作已提交"
2. 系统在后台执行
3. 完成后更新通知（成功/失败）
4. 页面自动刷新数据

### 6.3 错误处理

| 错误范围 | 表现 |
|---------|------|
| 整个页面加载失败 | 红色横幅在页面顶部，含"重试"按钮 |
| 单个操作失败 | 红色通知 + 相关区域显示错误文字 |
| 用户取消操作 | 灰色通知"已取消" |

### 6.4 危险操作确认

删除等不可逆操作会弹出确认对话框：

```
标题：确认操作
描述：具体说明这次操作的影响
[取消] [确认删除] ← 红按钮
```

背景有半透明遮罩，点击遮罩可取消。

### 6.5 操作完成后的刷新

每次安装/导入成功后，系统自动刷新页面数据，确保用户看到最新状态。

## 7. 产品概念模型

理解以下概念才能设计好这个页面：

### 7.1 Skill 是什么

一个 Skill 是指南文件（SKILL.md）+ 相关文件的集合，存储在中心仓库中。它可以安装到 agent 的工作空间，也可以导出到项目。

### 7.2 安装 vs 导入

| 操作 | 含义 |
|------|------|
| 安装 | 从外部来源（市场/Git/本地文件）把 skill 加到中心仓库 |
| 导入 | 把已存在于 agent 工作空间的 skill 注册到中心仓库 |
| 批量导入 | 扫描文件夹，自动发现并导入多个 skill |

### 7.3 安装来源

| 来源 | 特点 | 场景 |
|------|------|------|
| skills.sh 市场 | 社区共享，一键安装 | 发现和试用他人技能 |
| 本地文件夹/压缩包 | 离线，开发者自己的 skill | 开发/调试自己的 skill |
| Git 仓库 | 版本控制，可更新 | 从团队或开源项目安装 |
| Agent 工作空间（扫描发现） | 已存在但未纳入管理 | 整理已有的 skill |

### 7.4 已安装检测

系统会标记已安装的技能：
- 市场卡片上显示 "/" 标记
- Git 输入时提示"该仓库已安装"
- 扫描结果中标记"已导入"
- 防止重复安装

## 8. 三个 Tab 的关系

| Tab | 用户是谁 | 核心任务 | 心理模型 |
|-----|---------|---------|---------|
| 浏览市场 | 所有用户 | 发现、探索、尝鲜 | "逛街" |
| 本地安装 | 开发者 / 离线用户 | 安装自有或私有的 skill | "文件操作" |
| 从 Git 安装 | 开发者 / 团队用户 | 从版本库安装 | "克隆仓库" |

三个 Tab 完成同一件事（安装 skill），但入口不同，适配不同用户场景。

## 9. 视觉要点

### 9.1 颜色系统

- **强调色**（蓝色系）：主按钮、激活态、链接
- **成功色**（绿色）：已安装、成功通知
- **危险色**（红色）：删除、取消、错误通知
- **层级**：通过背景色深浅（页面背景 → 面板表面 → 悬停态 → 选中态）表现信息层级

### 9.2 间距与布局

- 页面最大宽度 1200px，居中
- 面板用统一圆角和边框
- 卡片网格用统一间距

### 9.3 反馈

- 所有可点击元素有悬停态（背景变深或文字变色）
- 所有按钮在 disabled 状态降低透明度
- 加载态用旋转图标
- Tab 切换用淡入动画
- 操作结果通过右下角通知反馈

## 基本原理（一句话说清）

这个页面管理的是一个**技能中心仓库**——就是一个本地目录，里面按文件夹存放所有 skill。

**安装的本质就是把 skill 文件复制到仓库里：**

| 来源 | 做了什么 |
|------|---------|
| 市场 / Git | 从 GitHub 克隆到临时目录，再复制进仓库 |
| 本地文件夹 | 直接把文件夹复制进仓库 |
| 本地压缩包 | 解压后复制进仓库 |
| 扫描发现 | 把 agent 工作空间里的 skill 注册（复制）到仓库 |

仓库里的每个 skill 就是一个目录，里面至少有一个 `SKILL.md` 指南文件 + 关联脚本/配置。

页面上的「已安装」标记，就是检查该 skill 的 source_ref 是否已在仓库中有记录。

> 整页面的核心交互：**选源 → 复制到仓库 → 刷新列表**。一切 UI 都是围绕这三步展开的。

### 市场的原理

市场本质上是 **skills.sh 这个第三方网站的前端橱窗**。整个市场功能就是三件事：

#### 1. 拉列表

页面从 skills.sh 拉取社区贡献的技能列表。有两种模式：

- **浏览模式**（无搜索）：拉取排行榜（All Time / Trending / Hot），按安装量排序
- **搜索模式**：用户输入关键词，查询匹配的技能

每次拿到的数据中，每条技能记录包含：
- 名称
- 来源标识（`GitHub用户名/仓库名` + `目录名`）
- 安装量

这个来源标识就是该技能在 GitHub 上的位置。

#### 2. 展示橱窗

列表以卡片网格展示，每张卡片告诉用户三件事：
- 这是什么技能（名称 + 描述）
- 谁做的（来源 + 头像）
- 别人用得多不多（安装量）

用户通过排行榜发现热门技能，通过搜索精准查找，通过 Source 筛选只看某个贡献者。

#### 3. 一键安装

用户点击安装按钮时，系统做的事：

```
1. 根据来源标识拼出 GitHub 仓库地址
2. 克隆该仓库到临时目录
3. 在仓库中找到对应的 skill 目录
4. 复制 skill 目录到本地中心仓库
5. 清理临时目录（避免残留）
```

所以

- [ ] 侧边栏 "安装" 导航项（图标 + 选中态高亮）
- [ ] 三段式 Tab 切换（市场 / 本地 / Git）
- [ ] 市场浏览：排行榜 + 搜索 + source 筛选 + 分页
- [ ] 技能卡片（头像 + 名称 + 来源 + 安装量 + 安装/已安装）
- [ ] 一键安装 + 进度反馈 + 取消
- [ ] 本地安装：文件夹 / 压缩包 / 批量导入
- [ ] 设备扫描 + 发现技能列表 + 重命名 + 逐个/全部导入
- [ ] Git 安装：URL 输入 + 重复检测 + 预览对话框
- [ ] 通知体系（进行中 / 成功 / 失败 / 取消）
- [ ] 危险操作确认对话框
- [ ] 成功通知含"查看"跳转
- [ ] 安装后自动刷新数据
- [ ] 空状态处理
- [ ] 国际化文本体系
